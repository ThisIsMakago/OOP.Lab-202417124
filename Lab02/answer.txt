Question 10: Java does not allow you to create a constructor to build a DVD by "title", and another to build by "category" because both title and category share the same data type (String). 
    In Java, method overloading is determined by the method signature (the method's name and its parameter types). 
    Both constructors would have the exact same signature: DigitalVideoDisc(String). 
    Therefore, the compiler cannot distinguish which constructor is being called, resulting in a compile-time error.
Question 14: Using varargs is preferable.
    It provides greater flexibility and a cleaner syntax. With varargs, the user can pass multiple objects directly separated by commas, or even pass an array if they want to. 
    On the other hand, if we use an array parameter (DigitalVideoDisc[] dvds), the user is forced to explicitly create and initialize an array every time they call the method, making the code more verbose.
Question 15: 
    1. Java is strictly a PASS-BY-VALUE language.
    2. After the calls of swap function, when passing objects to the swap method, Java copies the value of the references (memory addresses) of jungleDVD and cinderellaDVD into the local variables o1 and o2. 
    The swapping logic only swaps where these local variables point to inside the method. 
    The original references in the main method are completely unaffected and still point to their original objects.
    3. After the calls of changeTitle function, when jungleDVD is passed to changeTitle, the local variable dvd receives a copy of the reference. 
    At this point, both jungleDVD (in main) and dvd (in the method) point to the same object in memory. 
    When dvd.setTitle(title) is executed, it directly modifies the internal state of that shared object. 
    Even though the local variable dvd is reassigned to a new DigitalVideoDisc on the next line, the original object in the heap has already been mutated. 
    Thus, when we print jungleDVD in main, we see the updated title.
